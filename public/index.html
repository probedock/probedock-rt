Welcome to Probe Dock RT

Lightweight Probe Dock. It will help you in your daily work. Run tests, filter tests, get real time results in this interface.

The following instructions will remain on the screen until you receive your first test results.

Main user interface

Interface description

1. Summary

The summary consolidates all the results received by Probe Dock RT. You will see the following columns:

  • Version: Version of the project;
  • Project: The project API ID on Probe Dock if available. If not Any is used in place;
  • Category: The category refers to the type of tests;
  • Passed, Failed and Inactive: The status of the test results. In the case of Inactive  the test can be Passed or Failed;
  • All: The total of the test results for the category of test, in a project for the version;
  • Duration: The total duration of all the test results in the row.

The table is updated each time a result is received. Therefore, the table reflects a state that is always a reflect of your activity.

2. Squares / Tables

The squares view is a representation of the test results to get in one time the state of project in a version. You can click on squares to see the details.

The Any for project and version are placeholders to collect all test results that cannot be identified with a project API ID from Probe Dock.

3. Details

The details view allow you viewing the test result information. You should be able to see stack traces when a test is failing.

You will also have the possibility to view the following test data:

  • Key (k): If a test key is defined on Probe Dock and assigned to the test;
  • Fingerprint (fp): A unique identifier generated by the Probe Dock clients;
  • Duration: The duration of test execution;
  • Status: Failed, passed, inactive status of the test execution;
  • Project API ID and Version: General information about the project;
  • Name: The test name. This can be for example a humanization of class and method names in Java test;
  • Tags and Tickets: List of tags and tickets assigned to the test through the Probe Dock test decoration data.

Doing Alt + Click  on Key, Fingerprint, Name, Tag or Ticket will activate automatically the related execution filter.

You can close a test result details at any time by clicking on the cross in the top right corner. To open it again, you can click on the corresponding square in the test result squares.

4. View toolbar

This toolbar allow you to filter the data present in the interface. For example, you can show only the failing tests.

There is the list of controls you have at your disposal:

  • Filtering by status: You can filter the tests based on their status Inactive, Passed or Failed;
  • Switching the square vs. table view: The square view is a summary of the status of test results. In the table view, you will have more data available;
  • Clearing the view filters: You can reset the user interface filters. This will not clear the test results or the execution filters;
  • Text filter: The text filter can help to filter the test results based on their names or message content.
  • Reset: The reset button allows will clear all the test results. The execution filters will remains.

5. Execution toolbar

This toolbar allow you to filter the test at execution time. You define one or more filters that will be used by the Probe Dock clients at runtime to execute only specific tests.

You can filter the tests by: Key (key), Fingerprint (fp), code Name (name), Tag (tag), Ticket (ticket) or Generic (*).

To use the filters, you have just to enter the text in the text box. By default, the filter will be a Generic filter. To enter another filter, you have tofill the text box with this syntax: name:<text>. The filter types are in parenthesis in the previous paragraph.

Once a filter is configured, you can remove it by clicking on it or on the cross in the right of the filter.

6. Notifications

During the time the Probe Dock RT interface is open, several notifications will be received such the test run start/end or each test result. Also the status of the connection with the Probe Dock RT agent.

You can dismiss each notification by click on the cross on the top right corner or simply on the notification itself.

You can also acknowledge all the notifications at a time by clicking on the hammer on the top right corner of the notification area.

Finally, by clicking on a key or fingerprint in the test result notifications, you will open the corresponding test details.
